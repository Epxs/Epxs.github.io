Java Web(1)——Servlet

发表于 2020-03-18 | 分类于 Java Web

Servlet（Server Applet）是 Java Servlet 的简称，是一个用 Java 编写的服务器端程序。

Servlet 简介

Servlet 是运行在 Web 服务器或应用服务器上的程序，它是作为来自 Web 浏览器或其他 HTTP 客户端的请求和 HTTP 服务器上的数据库或应用程序之间的中间层。简而言之，Servlet 是运行在 Web 服务器端（Tomcat）负责与客户端进行通信的一组组件。

Servlet 的功能

Servlet主要有以下几点功能：

  • 接受客户端发来的请求数据。
  • 对请求进行处理，并可以与数据库进行通信。
  • 完成响应，返回基于客户端请求的动态HTML页面。

Servlet 生命周期

Servlet 生命周期可被定义为从创建直到毁灭的整个过程。 遵循以下过程：

  • 1 当浏览器访问 Servlet 的时候， Tomcat 会查询当前 Servlet 的实例化对象是否存在，如果不存在，则通过反射机制动态创建对象，如果存在，直接执⾏第 3 步。
  • 2 通过 init ⽅法完成初始化操作。
  • 3 通过 service ⽅法完成业务逻辑操作。
  • 4 关闭 Tomcat 时，会调⽤ destory ⽅法，释放当前对象所占⽤的资源。

Servlet 的⽣命周期⽅法调用顺序：⽆参构造方法 -> init() -> service() -> destory()。其中：

  • 1 ⽆参构造函数只调⽤⼀次，创建对象。
  • 2 init 只调⽤⼀次，用来初始化对象。
  • 3 service 可以调⽤ n 次，执⾏实际的业务⽅法。
  • 4 destory 只调⽤⼀次，负责销毁对象。

ServletContext 与 ServletConfig 对比

ServletConfig 和 ServletContext 的区别如下：

  • ServletConfig 作⽤于某个 Servlet 实例，每个 Servlet 都有对应的 ServletConfig。
  • ServletContext 作⽤于整个 Web 应⽤，⼀个 Web 应⽤对应⼀个 ServletContext，多个 Servlet 实例对应⼀个ServletContext。
  • ServletConfig 是局部对象，ServletContext 是全局对象。

Servlet的层次结构

实现Servlet有三种方式：

  • 实现 javax.servlet.Servlet 接口。
  • 继承 javax.servlet.GenericServlet 类。
  • 继承 javax.servlet.http.HttpServlet 类。

Servlet的继承结构如下图：

GenericServlet 实现 Servlet 接⼝，同时为它的子类屏蔽了不常用的⽅法，子类只需要重写 service 方法即可。

HttpServlet 继承 GenericServlet，根据请求类型进⾏分发处理，get 请求进⼊ doGet 方法，post 请求进⼊ doPost 方法。

开发者⾃定义的 Servlet 类只需要继承 HttpServlet，并重写 doGet 和 doPost 方法。这也是实际应用中常用的开发方式。
