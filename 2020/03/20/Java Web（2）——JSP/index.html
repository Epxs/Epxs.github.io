Java Web(2)——JSP

发表于 2020-03-20 | 分类于 Java Web

JSP 是 JavaWeb 服务器端的动态资源,它与 html 页面的作用是相同的，用于显示数据和获取数据。

JSP 概述

JSP 全称 Java Server Pages ，是一种动态网页开发技术。JSP 的本质是一种 Servlet，主要用于实现 Java web 应用程序的用户界面部分，并将最终的界⾯呈现给⽤户。

单纯的从代码的角度来看，JSP 文件实际上是一个注入了 Java 代码的 HTML 文件，是 HTML+CSS+JS+Java 的混合文件。

JSP工作原理

JSP 处理网页的步骤：

以下步骤表明了 Web 服务器是如何使用 JSP 来创建网页的：

  • 就像其他普通的网页一样，浏览器发送一个 HTTP 请求给服务器。
  • Web 服务器识别出这是一个对 JSP 网页的请求，并且将该请求传递给 JSP 引擎。通过使用 URL 或者 .jsp 文件来完成。
  • JSP 引擎从磁盘中载入 JSP 文件，然后将它们转化为 Servlet。这种转化只是简单地将所有模板文本改用 println() 语句，并且将所有的 JSP 元素转化成 Java 代码。
  • JSP 引擎将 Servlet 编译成可执行类，并且将原始请求传递给 Servlet 引擎。
  • Web 服务器的某组件将会调用 Servlet 引擎，然后载入并执行 Servlet 类。在执行过程中，Servlet 产生 HTML 格式的输出并将其内嵌于 HTTP response 中上交给 Web 服务器。
  • Web 服务器以静态 HTML 网页的形式将 HTTP response 返回到您的浏览器中。
  • 最终，Web 浏览器处理 HTTP response 中动态产生的HTML网页，就好像在处理静态网页一样。

Java 嵌入的三种方式

1 JSP 脚本：可以包含任意量的Java语句、变量、方法或表达式，用于执行Java的逻辑代码。

2 JSP 声明：可以声明一个或多个变量、方法，供后面的Java代码使用。

3 JSP 表达式：把 Java 对象直接输出到 HTML ⻚⾯中。

JSP 内置对象

JSP 内置对象是指 JSP 容器为每个页面提供的 Java 对象，我们可以直接使用它们而不需要显式声明。JSP 共有九大内置对象，分别是：

  • request：HttpServletRequest 类的实例，表示一次请求。
  • response：HttpServletResponse 类的实例，表示一次响应。
  • out：JspWriter 类的实例，用来写入内容，向浏览器输出数据。
  • pageContext：PageContext 类的实例，用来代表整个 JSP 页面。
  • session：表示⼀次会话，保存⽤户信息，HttpSession 的实例。
  • application：ServletContext 的实例，表示当前 Web 应⽤，全局对象，保存所有⽤户共享信息。
  • config：当前 JSP 对应的 Servlet 的 ServletConfig 对象，获取当前 Servlet 的信息，允许开发者访问 Servlet 或者 JSP 引擎的初始化参数。
  • page：当前 JSP 对应的 Servlet 对象， page 对象就是this对象的同义词。
  • exception：Exception 类的对象，代表发生错误的 JSP 页面中对应的异常对象。

HTTP 状态码

当浏览者访问一个网页时，浏览者的浏览器会向网页所在服务器发出请求。当浏览器接收并显示网页前，此网页所在的服务器会返回一个包含HTTP状态码的信息头（server header）用以响应浏览器的请求。常见的状态码及其表示的信息：

  • 200：OK，表示正常，请求成功。
  • 400：表示客户端请求的语法错误，服务器无法匹配。
  • 404：Not Found，服务器无法根据客户端的请求找到资源（网页）。
  • 500：服务器内部错误，无法完成请求，或程序抛出了异常。

JSP 页面重定向

转发请求先用 request 对象的 getRequestDispatcher() 方法获取一个 requestDispatchar 对象，再调用该对象的 forward() 方法。当需要将文档移动到一个新的位置时，就需要使用 JSP 重定向了。最简单的重定向方式就是使用 response 对象的 sendRedirect() 方法。重定向意味客户端向服务器发送一个新的请求。

请求转发和重定向主要有以下几点区别：

  • 转发是将同一个请求传给下一个JSP页面，重定向是向下一个页面发送一个新的请求。
  • 转发是同一个请求再服务器端进行传递，浏览器的地址栏不会发生改变，也叫服务器跳转。
  • 重定向由客户端发送⼀次新的请求来访问跳转后的⽬标资源，浏览器的地址栏发生改变，也叫客户端跳转。
  • 如果两个页面之间需要通过 request 来传值，则必须使⽤转发，不能使⽤重定向。
