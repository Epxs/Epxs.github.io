Java Web(3)——Session 与 Cookie

发表于 2020-03-21 | 分类于 Java Web

Session 和 Cookie 是 Web 开发中的两个重要概念，是会话技术的两种实现方式。

会话技术

会话技术：HTTP 是无状态的协议，这意味着每次客户端发出请求时，服务器不会记录下先前客户端请求的任何信息，即服务器⽆法识别每⼀次 HTTP 请求的出处（不知道来⾃于哪个终端）。所以必须要有⼀种技术来让服务器知道请求来⾃哪里，这就是会话技术。

以电商交易为例，当 A 要购买一件商品，向服务器发起“购买”请求，要求服务器将商品加到自己的购物车，因为服务器无法得知该请求的出处，所以就可能将商品加到 B（或C或D…）的购物车中。

会话：会话就是客户端和服务器之间发生一系列请求的过程，具体到实际情况中，可将会话看成浏览器从打开到关闭的一个过程。

会话的实现方式：

  • session：服务器会话技术
  • cookie：客户端会话技术

Session

session 作为 JSP 九大内置对象之一，是 HttpSession 类的实例。用来跟踪在各个客户端请求间的会话。可在一次会话的多次请求中共享数据，数据保存在 session 对象中。属于同一次会话的请求都有一个相同的标识符：sessionID。

Cookie

Cookie，客户端会话技术，服务端在 HTTP 响应中附带传给浏览器的⼀个⼩⽂本⽂件，⼀旦浏览器保存了某个 Cookie，在之后的请求和响应过程中，会将此 Cookie 来回传递，这样就可以通过 Cookie 这个载体完成客户端和服务端的数据交互。

Cookie 常用方法

  • void setMaxAge(int age) 设置 Cookie 的有效时间，单位为秒。
  • int getMaxAge() 获取 Cookie 的有效时间（值默认为-1）
  • String getName() 获取 Cookie 的 name。
  • String getValue() 获取 Cookie 的 value。

Session 和 Cookie 的区别

  • Session 保存在服务器端，保存的数据是 Object 类型，并随会话结束（浏览器关闭）而销毁，数据相对安全，一般用来保存重要信息。
  • Cookie 保存在客户端，保存的数据类型是 String 类型，可以长期保存在客户端，数据相对不安全。
  • 存取数据的方式：Session 用 setAttribute 存， getAttribute 取。Cookie 通过 response.addCookie(new Cookie(String name,String value)) 将数据发送到客户端，取数据时通过 request.getCookies() 得到一个 Cookie 数组。
