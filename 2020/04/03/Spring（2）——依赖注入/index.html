Spring(2)——依赖注入

发表于 2020-04-03 | 分类于 Spring

依赖注入，简称 DI，是实现 IoC 的一种方式。

依赖注入

当一个对象中包含了另一个对象，即对象之间有级联关系时，可通过设置 ref 属性的值，实现不同 bean 之间的关联。

通过 ref 属性将其他 bean 对象注入给当前 bean 对象的过程，叫依赖注入(DI)，Spring 框架中，IoC 和 DI 是密不可分的。

注入的方式

在 Java 中，在对象的成员赋值时，既可以通过 Setter 方法，也可以通过有参构造。同样，Spring 也提供了这两种方式来给 bean 注入属性。

Setter 方法

使用 property 标签装配 bean 时，会先调用无参构造函数，创建出一个对象，再通过调用 setter 方法，给属性赋值。所以，使用 property 装配 bean 时，实体类必须要有 Setter 方法与无参构造，否则便会报错。

1 修改 Student 类，方便查看方法调用的过程。

2 配置 student Bean。

3 取出 student 对象。

当去掉 setter 方法时，提示错误信息：“无效的 setter 方法”。

有参构造

constructor-arg 标签用于配置构造方法，要求所配置的对象必须含有有参构造函数。constructor-arg 标签有以下属性：

  • name ：参数的名称
  • value：设置普通数据
  • ref：引用数据，可关联到另外一个bean，一般是另一个bean的id值
  • index：指定参数的序号
  • type：指定参数的类型
