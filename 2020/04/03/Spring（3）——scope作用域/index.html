Spring(3)——Bean的作用域

发表于 2020-04-03 | 分类于 Spring

Spring 容器中的一个 Bean 有多种作用域。不同的 Bean 也可通过继承或依赖的方式关联。

Bean 的作用域

Spring 中的 bean 是根据 scope 来生成的，scope 表示 bean 的作用域。

scope 有以下四种类型：

  • singleton：单例，IoC 容器中配置的 bean 是唯一的，bean 以单例方式存在，每次获取的对象也是唯一的。
  • prototype：原型，每次从容器中获取 bean 时，都返回一个新的实例。
  • request：请求，在一次请求中有效。每次 HTTP 请求都会创建一个新的 bean。
  • session：会话，同一个 session 共享一个 bean，不同session 使用不同的 bean。

request 和 session 只适用于Web项目，scope 的默认值是 singleton。

Singleton 单例模式

bean 的作用域被默认为 singleton。

当一个 bean 的作用域为 singleton 时，IoC 容器中只会存在一个共享的 bean 实例，并且所有对该 bean 的请求，只要传入的 id 值与该 bean 定义的 id 相匹配，就只会返回同一个 bean 的实例。

prototype 原型模式

当一个 bean 的作用域为 prototype ，表示一个 bean 定义对应多个对象实例。prototype 作用域的 bean 会导致每次从 IoC 中获取对象时都会创建一个新的 bean 实例。每次获取到的对象不是同一个对象。

Bean 的继承

Java 中的继承关系指的是类层面的继承，子类可继承父类的成员变量和成员方法，而 Spring 中的继承针对的是对象层面，即 bean 与 bean 之间的继承。通过设置 parent 属性，可实现 bean 之间的继承。

Bean 的依赖

与继承类似，依赖关系也是 bean 和 bean 之间的一种关联方式，配置依赖关系之后，被依赖对象的创建一定先于依赖对象。

没有添加的依赖时，对象的创建顺序遵循 xml 文件中配置的先后顺序，即写在的前面的先创建。但给 bean 添加了依赖关系之后，被依赖的对象一定会先创建。
