Spring(1)——IoC概述

发表于 2020-04-03 | 分类于 Spring

Spring 是一个企业级开发框架。可以说，Spring 的出现，颠覆了 Java 行业的“游戏规则”。

Spring 概述

Spring 是一个开源框架，它由 Rod Johnson 创建。它是为了解决企业应用开发的复杂性而创建的。 Spring 使用基本的 JavaBean 来完成以前只可能由 EJB 完成的事情。然而， Spring 的用途不仅限于服务器端的开发。从简单性、可测试性和松耦合的角度而言，任何 Java 应用都可以从 Spring 中受益。Spring 是一个轻量级的控制反转（IoC）和面向切面（AOP）的容器框架。

Spring 框架结构如下图：

Spring 核心机制

  • IoC：Inverse of Control，即控制反转。
  • AOP：Aspect Oriented Program ，即面向切面编程。

使用Spring的优势

  • 低侵入，低耦合 （降低组件之间的耦合度，实现软件各层之间的解耦）
  • 声明式事务管理（基于切面和惯例）
  • 方便集成其他框架（如MyBatis、Hibernate）
  • 降低 Java 开发难度
  • Spring 框架中包括了 J2EE 三层的每一层的解决方案（一站式）

IoC 控制反转

IoC 概述

  • IoC 即控制反转，是一种程序设计过程中的设计思想。使用IoC 能够将程序中由开发者创建对象的控制权，交给 Spring 框架来管理。
  • 传统的设计模式：正向控制，如果要使用某个对象时，需由开发者手动去创建需要的对象。
  • 控制反转：若要使用某个对象，开发者无需手动去创建对象，只需从 IoC 容器中取出所需的对象即可。如此一来，开发者无需关心创建对象的过程，即实现了将创建对象的控制权反转给 Spring 。

举个例子，当我们需要某些物品时，传统的方式是自己去超市购买。从选购到付款，最后拿回家，这整个过程都由自己完成。这是一个主动“创造”，正向控制的过程。

根据 IoC 的思想，当我们需要某些物品时，我们只需要将所需的商品列出清单，交给一位代购，由代购进行采购，最后送到我们手里。这样一来，采购权便交给了代购，实现了“控制反转”。

总结一下，Spring 创建对象的步骤如下：

  1. 通过 xml 文件配置 Java bean。
  2. 在测试类中加载配置文件 spring.xml ，生成一个 applicationContext 对象。
  3. 调用 applicationContext 对象的 getBean() 方法，获取对象，参数为配置文件中 bean 标签的 id 值。

IoC底层原理

IoC的实现原理，其根本还是在于利用了java的反射机制。IoC容器创建一个对象的过程如下：

  • 1 解析 xml 配置文件，读取到类名。
  • 2 使用反射机制的API，基于类名实例化对应的对象实例。
  • 3 将对象实例，通过构造函数或者setter()方法，给属性赋值，最后将实例化好的对象返回。
