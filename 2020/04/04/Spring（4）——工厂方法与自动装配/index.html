Spring(4)——工厂方法与自动装配

发表于 2020-04-04 | 分类于 Spring

IoC 是典型的工厂模式，通过 IoC 容器能够更加高效的管理类工厂。

Ioc 工厂方法

IoC 作为典型的工厂模式，通过 IoC 工厂模式创建 bean 主要由两种方法：

  • 静态工厂方法
  • 实例工厂方法

3 传统的获取 computer 对象的方式是通过手动调用静态工厂方法来实现的。

4 在 Spring IoC 中，我们将 ComputerFactory 这个静态工厂类交给 IoC 容器来管理。在 xml 文件中，配置静态工厂。

5 取出 computer 对象。

实例工厂方法，顾名思义就是要先将工厂进行实例化，再通过调用工厂方法来创建对象。

1 先定义的一个实例工厂 InstanceComputerFactory，并提供获取 computer 对象的非静态方法。

2 传统方式

总结：静态工厂类中，初始化操作写在静态代码块里，在类加载的时候完成初始化操作，而定义的静态工厂方法可以通过类名直接调用，无需创建工厂对象，所以在配置文件中，只需配置一个 computer bean，而不需要配置 ComputerFactory bean。

实例工厂方法中，初始化操作写在构造函数里，且调用的是非静态方法，所以创建 computer 对象时，必须先创建工厂的实体类。所以配置文件中，必须要先配置一个实例工厂对象，再去配置 computer bean 。

IoC自动装载

IoC 可以完成对象的创建，通过设置 property 标签的ref属性可以实现依赖注入（DI）。这是一种手动注入的方法，而 Spring 框架也提供了一种自动注入的方式，开发者无需手动配置 property ，由 IoC 容器自动完成注入 bean 的操作，即自动装载（AutoWire）。

实现自动装载有两种方式：

  • byName：通过属性名
  • byType：通过数据类型

byName

1 创建一个 Student 类，Student 类的成员里包含一个 computer 对象。

2 配置 student bean 和 computer bean，通过 byName 的方式，进行自动装载实现依赖注入。

3 获取 student 对象

测试结果：

这里在配置 student 对象时，没有给通过 porperty 的 ref 属性给 computer 赋值，而是设置 autowire 的值为 byName，通过属性名自动装载，所以IoC容器会自动匹配属性名，去找一个 id=”computer” 的 bean，如果找到，就将这个 bean 自动注入到 student bean中。

这里要求 computer bean 的 id 值必须和 Student 类中的属性名一致，否则将注入失败。

byType

1 配置 student bean 和 computer bean，并通过 byType 的方式，进行自动装载。

2 获取 student 对象

测试结果：

通过 byType 的方式进行自动装载时，IoC 容器会自动去匹配一个类型是 Computer 的 bean，与该 bean 的 id 值无关，只要类型匹配，IoC便会将该 computer bean 注入到 student 中。

ByType 的方式要求整个 xml 文件中，只能有一个 computer bean，如果配置了多个 computer bean，IoC 容器便不知道应该将哪一个 bean 注入给当前 student，程序便会报错。
