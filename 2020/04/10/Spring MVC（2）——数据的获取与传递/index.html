Spring MVC(2)——数据的获取与传递

发表于 2020-04-10 | 分类于 Spring MVC

接收请求的输入和传递模型数据是 Spring MVC 的一个重要功能，是数据在前后端进行传递的重要保障。

@RequestMapping 注解

@RequestMapping 注解用于将请求映射到方法上，该请求也可以与类绑定。当控制器在类级别上添加 @RequestMapping 注解时，该注解会应用到控制器的所有处理器方法上。

该注解包含以下属性：

  • value：指定所要处理的请求路径。当参数中只定义 value 时，可不声明。例如：@RequestMapping(“/“) 和 @RequestMapping(value =”/“) 是等价的。
  • method：指定请求类型。
  • params：指定请求中必须包含哪些参数。

对参数进行约束

如果形参名和传来的参数名不一致，或者需要形参进行相关设置，可通过 @RequestParam 注解。有以下属性：

  • value：将请求中的参数和当前形参映射起来，值为参数名。
  • defaultValue：默认值，当没有接收到这个参数时，为形参设定一个默认值。
  • required：设定参数是否为必填参数。

这里将 name 和 username ，age 和 userage 映射起来，并为其设定了默认值。

当没有传入参数时，输出默认值。

当传入 name 和 age 后，输出 name 和 age 。

传递模型数据到视图

一套完整的数据交互不仅要接受前端传来的参数，还需要从后端将数据返回给前端。Spring MVC 能将模型数据传递到视图中，即由控制器获取获取模型数据，再由 JSP 页面进行数据展示。

控制器会将获取到的模型数据绑定到 JSP 域对象中。

Spring MVC 提供了以下几种方式添加模型数据：

  • Map
  • Model
  • ModelAndView
  • @ModelAttribute

Map

Spring MVC 在内部使用 Model 接口存储模型数据，在调用业务方法前会创建一个隐含对象作为模型数据的存储容器。只需要在业务方法中定义 Map 类型的形参，方法体中便可通过对 map 的操作来完成模型数据的添加。
