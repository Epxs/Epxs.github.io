Spring(6)——AOP

发表于 2020-04-07 | 分类于 Spring

面向切面编程是对面向对象编程的补充，也是 Spring 体系中的一套重要机制。

AOP概述

AOP 是 Spring 框架中除 IoC 之外的另一套重要机制。

AOP 全称为 Aspect Oriented Programming，面向切面编程。面向切面编程（AOP）实际上是对面向对象编程（OOP）的补充和延伸。利用AOP可以使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

举个例子，现在我们定义了多个业务方法，每个业务方法除了执行各自的业务逻辑之外，还要求在方法开始执行之前和方法执行完毕之后，打印一些日志信息（打印日志信息的代码几乎完全相同）。如下图：

这里有三个业务方法，这三个方法的耦合程度是比较高的，打印日志这个操作出现了很多次，如果反应到代码中，也就是需要将“打印日志”这段代码重复写很多次，这会使得程序的耦合度提高，不仅降低了程序的可读性，更不利于后期的维护。

我们能否将这段代码提取出来，进行统一集中的管理，从而让每个方法只需专注于自己的业务逻辑操作呢？答：可以。

我们将原本纵向排布的业务方法“横切一刀”，抽取出一个“切面”，将这些重复的代码写在这个切面对象里，对这个“切面”编程。这个过程如下图：

AOP 相关术语

  • target：目标类，需要被代理的类（被切的类）。
  • joinpoint：连接点，指那些可能被拦截到的方法。
  • pointcut：切入点，需要切入的地方，通过切入点可定位到连接点。
  • advice：通知，对面对象完成的某些具体操作（如上面的打印日志）。
  • aspect：切面，抽取出来的切面对象，是切入点pointcut和通知advice的结合。

AOP通知类型与连接点

  • before：前置通知，在目标类方法执行之前执行。
  • after：最终通知，在目标类方法调用过后执行。
  • after-throwing：异常通知，在目标类方法抛出异常时执行。
  • after-returning：后置通知，在目标类方法正常返回后执行。
  • around：环绕通知，在环绕通知中，目标类方法并不会执行，需要手动调用，使用环绕通知，可以让开发者手动控制通知的执行时机。
