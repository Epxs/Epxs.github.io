Spring(5)——基于注解的开发

发表于 2020-04-05 | 分类于 Spring

Spring 提供了两种装配 bean 的方式，一种基于 xml 文件配置 bean，另一种则是使用注解。

使用注解的方式配置 bean 可以简化开发过程，其目的与 xml 文件配置 bean 一样，都是为了降低代码的耦合程度，只是二者的形式不一样而已。

使用注解配置 bean 前，需要开启注解扫描，将指定包下使用了注解的类扫描到 IoC 容器中。开启扫描需要在 xml 文件中如下添加配置。

使用注解装配 bean

用于装配 bean 的注解主要有以下几种：

  • @Component：在类的上面使用该注解，相当于在 xml 中配置了一个bean。
  • @Controller：用于 controller 层的注解。
  • @Service：用于 service 层的注解。
  • @Repository：用于 repository 层的注解。

@Controller，@Service，@Repository 用法和功能与 @Component 一样，不同的业务层使用不同的注解，可以使代码结构更加规范，逻辑更加清晰。

@Autowired

@Autowired 可以自动按类型注入，即自动装载，默认方式是 byType 。

@Qualifier

@Qualifier 可以在自动注入的基础上，再指定 bean 的 id ，一般和 @Autowired 结合使用。

使用 IoC 注解实现分层架构

程序分层：再应用开发中一般将后台程序分为3层，分别是 Controller 层，Service 层，DAO 层。

这三者的关系是，Controller 层调用 Service 层，Service 层调用 DAO 层（Controller -> Service -> DAO）。Service 层和 DAO 层设计接口和实现类。现在通过 Spring 注解的方式，来实现对程序的分层。
