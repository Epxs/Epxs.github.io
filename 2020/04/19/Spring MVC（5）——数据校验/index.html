Spring MVC(5)——数据校验

发表于 2020-04-19 | 分类于 Spring MVC

用户的输入是很不确定的，为了保证数据的合法性和正确性，往往需要对数据进行校验。

数据校验的两种方式：

  • 客户端验证：在前端可以对用户输入的数据进行合法性验证，一般通过 JavaScript 实现，这是第一道防线。
  • 服务器验证：只做前端校验是不安全的，当数据传到后端时，往往需要再进行一次校验，建立第二道防线。

Spring MVC 可通过 Validator 接口和 JSR 303 两种方式完成数据在服务端的二次校验。

JSR 303

Spring 支持对 Java 校验 API （JSR 303）的支持，JSR 303 是一个标准，使用 JSR - 303 标准校验数据时不需要定义验证器，而是通过注解的方式可以直接在实体类中添加每个属性的验证规则。

JSR 303 标准提供了以下注解：

  • @Null：被注解的元素必须为 null。
  • @NotNull：被注解的元素不能为 null。
  • @AssertFalse：被注解的元素必须为 boolean 类型，且为 false。
  • @AssertTrue：被注解的元素必须为 boolean 类型，且为 ture。
  • @Max：被注解的元素必须是数字，并且值要小于等于给定的值。
  • @Max：被注解的元素必须是数字，并且值要大于等于给定的值。
  • @Digits：被注解的元素必须是数字，并且必须有指定的位数。
  • @Past：被注解的元素必须是一个已过去的日期。
  • @Future：被注解的元素必须时一个将来的日期。
  • @Size：被注解的元素必须是 String，集合或数组，且长度要符合给定的范围。
  • @Pattern：被注解的元素符合给定的正则表达式。
  • @Email：被注解的元素必须是一个电子邮箱。
  • @Length 被注解的元素的大小必须在指定的范围内。
  • @NotEmpty 被注解的字符串串的值必须非空。
