Spring Cloud实战（一）——服务的注册与发现

发表于 2020-10-14 | 分类于 Spring Cloud

微服务架构中，服务的治理和监控是关键。

服务注册中心

什么是服务注册中心

在微服务架构中，业务被拆分了一个一个的服务，所有这些服务既是独立的，同时又是相关联的。一个服务既要保证自身业务的正常完成，同时还要去发现其他服务，并和其他服务进行通信。当服务越来越多，服务的治理和监控就变得尤为重要。
所谓服务注册中心，其本质也是一个服务，只是这个服务不完成任何业务功能，仅完成对整个微服务体系的服务注册和服务发现，并对服务进行健康监控和统一治理。

服务注册中心应当完成以下功能：

  1. 完成服务的注册，生成服务注册表用于存储服务的基本信息，如服务名，端口，ip地址等。
  2. 完成服务的发现，当进行服务的调用时，能够查询可用的服务列表及其网络信息，完成调用。
  3. 对服务进行健康监控，当发现一个服务不可用时，能及时对该服务进行移除。

常用服务注册中心及其区别

Spring Cloud 作为一个微服务的组件集合，整合了多种服务注册中心，如：Zookeeper，Eureka，Consul 和 Nacos 等。

CAP定理：在一个分布式系统中，一致性（Consistency）、可用性（Availability）、分区容错性（Partition tolerance）。CAP 原则指的是，这三个要素最多只能同时实现两点，不可能三者兼顾。

  • 一致性C：分布式系统中的所有结点的数据在同一时刻保持一致。
  • 可用性A：系统提供的服务必须一直处于可用状态，客户端一直可以正常访问并得到系统的正常响应。
  • 分区容错性P：分布式系统中的某个节点或者网络分区出现了故障的时候，整个系统仍然能对外提供服务。也就是说部分故障不影响整体使用。

使用 Eureka 搭建注册中心

Eureka Server

1 创建一个 Spring Cloud 工程，引入 Eureka Server依赖。

pom.xml

2 配置文件中进行注册中心的相关配置。

application.properties

由于服务注册中心（EurekaServer） 其本身也是一个服务，也可以注册到注册中心并从注册中心获取其他服务的信息。默认情况下，注册中心会自己注册自己。eureka.client.register-with-eureka=false 表示关闭其自我注册。eureka.client.fetch-registry=false 表示当它作为一个服务客户端时，不去从其他的 EurekaServer 获取信息。

3 启动类上开启 EurekaServer。

4 启动项目，访问 http://localhost:8761，看到如下页面，服务注册中心搭建完成，当前没有任何服务。

1 创建一个 Spring Cloud 工程，添加 Eureka Client 依赖。

pom.xml

2 配置文件添加相关配置。

application.properties

3 启动类开启 EurekaClient。

4 启动 EurekaServer 和 EurekaClient ，访问 http://localhost:8761，发现该服务已经注册到注册中心。而业务代码的开发，和在 Spring Boot 项目中就是一样的了。

使用 Consul 搭建注册中心

Consul 安装

Consul 是一个可以提供服务发现的发现，健康检查的分布式组件。和 Eureka 不同的是，使用 Consul 作为注册中心，不需要我们编写任何的注册中心的代码，其本身就是一个可运行的文件。

1 下载 Consul。地址：https://www.consul.io/downloads。

2 解压。使用 cmd 运行

3 使用 consul -v 可查看 consul 版本。

4 输入 consul agent -dev ，开启 consul。


5 访问 http://localhost:8500，看到如下页面，Consul 注册中心搭建完成。

Consul Client

Consul 注册中心搭建完成，接下来再搭建一个 Consul 客户端的微服务。

1 创建 Spring Cloud 工程，引入 consul 客户端依赖。由于 Consul 依赖 spring actuator 进行健康检查，所以还需引入 ‘’spring-boot-starter-actuator’’。

pom.xml

2 配置文件添加相关配置。

application.properties

3 启动类开启 ConsulDiscoveryClient。

4 启动 Consul 注册中心和 ConsulClient，访问 http://localhost:8500，服务注册成功。
