Java 多线程之 Thread 类常用方法

发表于 2020-07-10 | 分类于 Java SE

Thread 类是 Java 多线程中的一个重要的类，本文将介绍 Thread 类中一些用于管理线程的常用方法。

线程与进程

  • 进程：指在系统中正在运行的一个应用程序；程序一旦运行就是进程；进程——资源分配的最小单位。
  • 线程：系统分配处理器时间资源的基本单元，或者说进程之内独立执行的一个单元执行流。线程——程序执行的最小单位。

简而言之：一个正在运行的程序就是一个进程，一个进程中可以有多个线程。

线程的状态

线程的五大状态分别为：创建状态（New）、就绪状态（Runnable）、运行状态（Running）、阻塞状态（Blocked）、死亡状态（Dead）。线程的生命周期如下：

  • 创建状态：单纯定义一个线程，此时线程已经拥有了内存空间，但还不可运行。
  • 就绪状态：线程启动后，进入就绪状态，等待 CPU 调度。
  • 运行状态：线程获得 CPU 调度，占用 CPU，开始执行。
  • 阻塞状态：线程开始运行后，由于各种原因，如等待资源分配，线程休眠 (Sleep) 等。不能继续执行下去，此时线程进入阻塞状态。阻塞结束后，重新进入就绪状态，等待 CPU 下一次调度。
  • 死亡状态：线程调用结束，处于死亡状态，线程死亡之后不可再调用。

Thread 类常用方法

Thread 类中有一些管理线程的方法，这些方法可以用于控制线程的执行。

线程休眠 sleep

线程休眠：指的是让线程暂缓执行，等到预计时间之后再恢复执行。

  • 线程休眠会交出 CPU，让 CPU 去执行其他的任务。
  • 调用 sleep() 方法让线程进入休眠状态后，sleep() 方法并不会释放锁，即当前线程持有某个对象锁时，即使调用 sleep() 方法其他线程也无法访问这个对象。
  • 调用 sleep() 方法让线程从运行状态转换为阻塞状态
  • sleep() 方法调用结束后，线程从阻塞状态转换为就绪状态。

sleep(Long millis) 可传入一个 millis 毫秒值，即休眠多少毫秒。

线程让步 yield

线程让步：暂停当前正在执行的线程对象，并执行其他线程。

  • 调用 yield() 方法让当前线程交出 CPU 权限，让 CPU 去执行其他线程。
  • yield() 方法和 sleep() 方法类似，不会释放锁，但 yield() 方法不能控制具体交出 CPU 的时间。
  • yield() 方法只能让拥有相同优先级的线程获取 CPU 执行的机会。
  • 使用 yield() 方法不会让线程进入阻塞状态，而是让线程从运行状态转换为就绪状态，只需要等待重新获取 CPU 执行的机会。

线程让步是否成功，取决于 CPU 的执行调度，即调用 yield() 方法后，不一定能成功让步，还要看当前 CPU 的运行状态。
