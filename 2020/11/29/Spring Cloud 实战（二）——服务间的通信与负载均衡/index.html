Spring Cloud实战（二）——服务间的通信与负载均衡

发表于 2020-11-29 | 分类于 Spring Cloud

在单体应用中，各模块之间的调用是通过编程语言级别的方法或者函数来实现的。而基于微服务的分布式应用是运行在多台机器上的，服务之间又该如何通信呢?

服务间的通信方式

上篇文章中，我们搭建出了一个 Spring Cloud 的基础架构，完成了服务的注册与发现。接下来，我们的就需要重点关注各个服务之间如何进行通信，即服务间如何互相调用。

在 Spring Cloud 中，微服务的通信方式是通过 HTTP REST 的方式进行的。主要有以下两种方式：

  • RestTemplate + Ribbon
  • Feign

RestTemplate 服务调用

简介

RestTemplate 是从 Spring3.0 开始支持的一个 HTTP 请求工具，它提供了常见的 REST 请求方案的模版，例如 GET 请求、POST 请求、PUT 请求、DELETE 请求以及一些通用的请求执行方法 exchange 以及 execute。

Spring 框架提供的 RestTemplate 简化了服务间的通信方式，相较于之前常用的 HttpClient 等 http 通信工具，RestTemplate 是一种更优雅的调用 RESTful 服务的方式。

使用 Ribbon 负载均衡

问题

上述使用 RestTemplate 进行服务调用的例子中，还存在以下问题：

  • 在微服务项目中，肯定会有多个 product 服务实例并对应多个 url，而 customer 服务中调用 product 的 url（localhost:8081/product/getAll）是写死的，这样不利于流量的分发，以及代码的维护。
  • 当某个 product 服务宕机，必定也会导致某个 customer 服务不可用。

什么是负载均衡？

负载均衡（Load Balancer）是指把用户访问的流量，通过某种转发策略，均匀的分发到多台服务器上，每台服务器就可以独立的响应和处理请求，从而实现分散负载的效果。负载均衡技术提高了系统的服务能力，增强了应用的可用性。

总结：RestTemplate 没有实现负载均衡。

2 在 customer 服务的 RestTemplate 配置中通过 @LoadBalanced 注解，给 restRemplate 开启负载均衡的功能。

Feign 服务调用

简介

Feign 是一个声明式的伪 Http 客户端，它使得写 Http 客户端变得更简单。使用 Feign，只需要创建一个接口并注解。它具有可插拔的注解特性(可以使用 Springmvc 的注解)。Feign 默认集成了 Ribbon，默认实现了负载均衡的效果。

简而言之：

  • Feign 采用的是基于接口的注解
  • Feign 整合了ribbon
