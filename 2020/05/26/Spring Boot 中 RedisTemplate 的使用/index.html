Spring Boot 中 RedisTemplate 的使用

发表于 2020-05-26 | 分类于 Spring Boot

Spring Data Redis 是 Spring 大家族的一部分，提供了在 Spring 应用中访问和操作 Redis 的实现。

Redis 概述

Redis 简介

Redis 是一种特殊类型的数据库，是 NoSQL 非关系型数据库里面的一大重要产品。与传统的关系型数据库不同，Redis 彻底抛弃了“数据表”的概念，数据基于内存进行存储，并采用 key-value 的存储形式（有点类似 Java 中 HashMap）。

Redis 基本数据类型

Redis 中有五种基本数据类型，分别是：

  • String 字符串
  • List 列表
  • Set 集合
  • ZSet 有序集合
  • Hash 哈希

Redis 序列化

在使用 RedisTemplate 进行数据存储时会遇到一个问题，数据在 Java 程序中可以正常的取出，但当我们去 redis-cli 中去取对应的数据时却无法取出，存入的 key 和 value 值也变成了一串奇怪的字符。这是因为数据在 Redis 中存储与取出时，存在一个序列化与反序列化的过程。

1 向 Redis 中存入一个 username。

2 redis-cli 无法取出正常的值。

Spring Date Redis 提供了多个序列化器，分别是：

  • StringRedisSerializer：对 String 数据进行序列化。
  • JacksonJsonRedisSerializer：用 Jackson1，将对象序列化成 JSON。
  • Jackson2JsonRedisSerializer：用 Jackson2，将对象序列化成 JSON。
  • JdkSerializationRedisSerializer：使用 Java 序列化。
  • GenericToStringSerializer：使用 Spring 转换服务进行序列化。
  • OxmSerializer：使用 SpringO/X 映射的编排器和解排器实现序列化，用于XML序列化。

RedisTemplate 默认使用 JdkSerializationRedisSerializer 的序列化方式，会形成类似 “\xac\xed\x00\x05t\x00\busername” 这样的结果。而 StringRedisTemplate 则会使用 StringRedisSerializer，不会有 “\xac\xed\x00\x05t\x00\” 这样的前缀，直接可以生成 username。

使用 StringRedisTemplate 则不会出现上述问题。
