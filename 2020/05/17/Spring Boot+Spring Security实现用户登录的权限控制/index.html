Spring Boot + Spring Security 实现用户登录的权限控制

发表于 2020-05-17 | 分类于 Spring Boot

应用程序的安全问题是重中之重，Spring Security 为 Web 程序提供了一套完成的安全解决方案。

引入

在传统的 Web 开发，如果要过滤一个用户请求，通常是使用过滤器或者拦截器。如：对一个前端传过来的用户请求，我们可以在过滤器中进行某些操作，来判断是否允许将该请求继续向下传递。

这里定义了一个简单的过滤器，当判断当前 Session 中如果存在用户信息（说明已经登陆），则允许对 “/resource” 的访问，实现一个简单的权限控制。

Spring Security 简介

Spring Security 是为基于 Spring 的应用程序提供安全保护的安全框架。Spring Security 提供了完整的安全性解决方案，他能够在 Web 请求级别的方法调用级别处理身份认证和授权。Spring Security 充分利用了 Spring 框架的面向切面（AOP）技术，最大程度的减少了代码侵入。

Spring Security 从两个角度来解决安全问题，它使用 Servlet 规范中的 Filter 来保护 Web 请求并限制 URL 级别的访问。Spring Security 还能够使用 Spring Aop 保护方法的调用——确保只有具备适当权限的用户才能访问安全保护的方法。

控制用户权限

不幸的是，这个程序并不是对所有用户开放的，我们需要限制一些用户对某些资源的访问。如普通用户（user）只能访问 Normal 页面，会员（vip）可以访问 Normal 和 Vip 页面，而管理员（admin）则有权限访问所有页面。

使用 Spring Security 来进行权限控制时，无需修改原来的代码，只需配置一个 Config 类，让该类继承 WebSecurityConfigurerAdapter ，在这个配置类中，完成与安全相关的配置。

定义一个 SecurityConfig 的配置类。

在 SecurityConfig 类中，通过重写 WebSecurityConfigurerAdapter 的 configure() 方法来配置 Web 安全。configure() 有三个重载：

  • configure(WebSecurity) 配置 Spring Security 的 Filter 链。
  • configure(HttpSercurity) 配置如何拦截请求。
  • configure(AuthenticationManagerBuilder) 配置用户存储。

配置用户存储

通过重写 configure(HttpSercurity) 方法，我们给不同的请求配置了不同的“角色”，仅仅也只是配置一个“角色”而已。如果“角色”不符，Spring Security 则会拦截相应的请求。

要让程序正常执行，还需要配置用户存储，即给每一个“角色”绑定用户。继续配置 SecurityConfig，并重写 configure(AuthenticationManagerBuilder) 方法。用户存储既可以从内存中读取，也可以从数据库中读取。这里选择从内存中读取。

如此一来，一个简单的安全配置便完成了，登陆不同的用户，可以拥有不同的访问权限。如登陆 vip ，输入密码 321 时，可以访问 Normal 和 Vip 页面，而无法访问 Admin 页面。
