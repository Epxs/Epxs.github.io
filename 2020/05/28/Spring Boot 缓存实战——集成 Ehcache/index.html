Spring Boot 缓存实战——集成 Ehcache

发表于 2020-05-28 | 分类于 Spring Boot

缓存（Caching）可以存放经常会用到的信息，当对同一个数据进行频繁的访问时，使用缓存能够大大提高程序效率。Spring Boot 也提供了对缓存的支持。

Spring 缓存抽象

简介

Spring 对缓存的支持有两种方式：

  • 注解驱动的缓存（推荐使用）
  • XML 声明式的缓存

Spring 使用缓存管理器（CacheManager）来对多个流行的缓存实现进行集成，支持 Ehcahe，Redis 等缓存技术。Spring Boot 默认使用 ConcurrentMapCacheManager 来实现缓存，我们也可以将其进行替换。

在 Spring Boot 应用中，我们已经无需手动去配置缓存管理器了，这些都已经由 Spring 自动完成。我们只需在使用相应的注解，就可以完成对缓存的操作。

Ehcache 简介

Ehcache 是一个基于Java实现的开源缓存管理库，提供了用内存、磁盘文件存储、以及分布式存储等多种灵活的管理方案。使用方式和原理都有点类似于 Spring事务管理，配合各项注解可以很容易的上手。

实现缓存

环境搭建完成后，我们启动 Spring 注解驱动的缓存，Spring 提供了以下缓存注解：

  • @Cacheable：表明在调用方法前，首先会在缓存中查找当前方法的返回值，如果找到，则返回缓存的值，不调用方法。否则，这个方法就会被调用，并将方法的返回值存入缓存。
  • @CachePut：在调用方法前，不去检查缓存，方法始终会被调用，并将方法的返回值存入缓存。
  • @CacheEvict：表示清除相应缓存。
  • @Caching：同时指定多个缓存注解。
